DB design for GUVI ZEN CLASS:

DATABASE SCHEMA:
***************

1.USERS TABLE:

   -USER_ID(PRIMARY KEY)
   -FIRST NAME
   -LAST NAME  
   -EMAIL
   -USERNAME
   -PASSWORD
   -ROLE(student,mentor,admin)

2.CLASSES TABLE:

   -CLASS_ID(PRIMARY KEY)
   -CLASS NAME
   -DESCRIPTION
   -MENTOR_ID(FOREIGN KEY TO USERS)
   -SESSIONS
   -TASKS
   -TASK URLS

3.STUDENTS TABLE:

   -STUDENT-ID(PRIMARY KEY)
   -USER_ID(FOREIGN KEY TO USERS)
   -EMAIL(FOREIGN KEY TO USERS)
   -PHONE
   -ATTENDANCE
   -FEEDBACK
   -ASSESMENTS
   -TASKS
   -DEAD LINE

4.ASSIGNMENTS TABLE:

   -ASSIGNMENT_ID(PRIMARY KEY)
   -STUDENT-ID(FOREIGN KEY TO STUDENTS)
   -TITLE
   -DESCRIPTION
   -DUE DATE

5.SUBMISSIONS TABLE:
   
   -SUBMISSION_ID(PRIMARY KEY)
   -ASSIGNMENT_ID(FOREIGN KEY TO ASSIGNMENTS)
   -USER_ID(FOREIGN KEY TO USERS)
   -SUBMISSION DATE
   -FILE(FOR ATTACHING SUBMITTED WORK)

6.DASHBOARD TABLE:
  
   -STUDENT-ID(FOREIGN KEY TO STUDENTS)
   -ATTENDANCE
   -CODEKATA
   -WEBKATA
   -SUBMITTED TASKS
   -PENDING TASKS

7.TASK TABLE:
   
   -STUDENT-ID(FOREIGN KEY TO STUDENTS)
   -TASK TITLE
   -SUBMISSION DATE
   -TASK SCORE
   -STUDENT COMMENTS
   -MENTOR COMMENTS

8.QUERY TABLE:

    -QUERY NUMBER
    -QUERY TITLE
    -DATE
    -STATUS

9.LEAVE APPLICATION TABLE:

    -STUDENT-ID(FOREIGN KEY TO STUDENTS)
    -FROM DATE
    -TO DATE
    -APPROVED
    -REASON

10.MOCK INTERVIEW TABLE:

    -STUDENT-ID(FOREIGN KEY TO STUDENTS)
    -MOCK INTERVIEW ROUND
    -INTERVIEW DATE
    -OVERALL SCORE
    -RECORDING URL
    -COMMENTS
    -ACTION

11.LEADERBOARD TABLE:

    -STUDENT-ID(FOREIGN KEY TO STUDENTS)
    -RANK
    -STUDENT NAME
    -BATCH
    -LEARNING


    => This is a basic structure that allows us to manage
       users,classes,students,assignments,submissions,dashboard,task,query,leave application,mock interview and leaderboard tables.
